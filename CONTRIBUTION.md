# Project Task Allocation & Responsibilities

## Project: LEARNIT – Student Learning Platform

To ensure clarity, accountability, and effective collaboration, the
project tasks are divided among team members based on roles and areas
of expertise.

---

## Margaret – Backend & System Architecture Lead

### Responsibilities

- Overall system architecture design
- Backend development using Node.js
- REST API design and implementation
- Authentication and authorization
- Role-based access control (Student, Instructor, Admin)
- Database design and PostgreSQL schema modeling
- API security, validation, and error handling
- Backend technical documentation

### Functions

Core system logic, data flow, security enforcement, and access control.

---

## Godwill – Frontend Lead

### Responsibilities

- Frontend development using Next.js and TypeScript
- User interface design and implementation
- Student, Instructor, and Admin dashboards
- Integration of frontend with backend APIs
- Client-side validation
- User experience and usability improvements

### Functions

User interaction layer and presentation logic.

---

## Vin-Wilson – Media, Testing & Documentation Lead

### Responsibilities

- Cloudinary integration for images, videos, and documents
- Media upload and management features
- Basic functional system testing
- Sample data creation (users, courses, content)
- Project documentation and reporting

### Functions

Media handling, testing, documentation, and system validation.

---

## Conclusion

Each team member is responsible for their assigned area and collaborates
through clearly defined interfaces such as APIs and specifications. This
division of responsibilities minimizes overlap, improves efficiency,
and ensures accountability throughout the development process.
